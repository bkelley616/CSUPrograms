Project6: An Improved Shell (Due at 11:59:59pm on 12/05/2019 (EST))

Group Arrangment.

Based on the shell program you have developed in project5, you are asked to improve your shell to support several I/O redirection. You are expected to use C programming language. Your implementation must work on Linux machine in FH131 lab.


I. Extra Features:
  1. (50 points) Support pipeline (|). Given the following command, 
    $ MYSHELL: cmd1 arg1 arg2 ... | cmd2 arg1 arg2 ... | cmd3 arg1 agr2 ...
    the standard output of cmd1 is connected to the standard input of cmd2, and the standard output of cmd2 is connected to the standard input of cmd3.
    • You may want to use pipe() system call.
    • You may assume at most two pipeline operators (|) in my test command while grading. Actually, it is easy to remove this assumption and your shell should handle unlimited number of "|"s.
  2. (20 points) Support Input Redirection. Your shell should support input redirection. In particular, if the following command is entered: 
    $ MYSHELL: cmd1 arg1 arg2 ... < myfile
    The input of process cmd1 will be redirected to a local file named "myfile".
  3. (20 points) Support Output Redirection. Your shell should support output redirection. In particular, if the following command is entered: 
    $ MYSHELL: cmd1 arg1 arg2 ... > myfile
    The screen output will be redirected to a local file named "myfile". This redirection capability should work for all commands in your shell.
  4. Bonus: (10 points) You will earn the bonus points if your shell supports the combination of pipeline, input and output redirection. For example: 
    $ MYSHELL: cmd1 arg1 arg2 ... < file1 | cmd2 arg1 ... | cmd3 arg1 ... > file2
  5. Note1: the above arguments: arg1, arg2, and etc, are optional. Your shell should not be restrictive on how many arguments being supported.
  6. Note2: the above extension only works with non-internal commands, and should not work with internal commands: cd, path and quit.
